# 3. Функциональные требования

## 3.1 FR-1: Загрузка IDS-файлов
**Описание**: Система предоставляет возможность загрузить один или несколько валидных файлов IDS.

**Вход**: Один или более файлов `.ids` (XML)
**Выход**: Распарсенная структура всех IDS в памяти

**Ссылки**: [[04_interfaces](04_interfaces.md#UI-1), [[05_constraints](05_constraints.md#TECH-3)

---

## 3.2 FR-2: Анализ требований IDS
**Описание**: Система идентифицирует все требования IDS, которые могут относиться к свойствам объектов (PSet).

**Вход**: Распарсенные IDS
**Выход**: Список требований, сгруппированных по `entity` + `propertySet`

**Алгоритм**:
1. Извлечение всех `specification`
2. Для каждой — анализ `applicability` (entity)
3. Для каждого `requirement` типа `property` — извлечение:
   - `propertySet` (имя набора)
   - `baseName` (имя свойства)
   - `dataType` (тип данных IFC)
   - `cardinality` (required/optional)
   - `value` (ограничения, если есть)

**Ссылки**: [[06_rules](06_rules.md), [[07_ifc_mapping](07_ifc_mapping.md)

---

## 3.3 FR-3: Предпросмотр результатов
**Описание**: Система отображает найденные PSet для подтверждения перед генерацией.

**Вход**: Список требований из FR-2
**Выход**: Древовидная структура: Entity → PSet → Свойства

**Ссылки**: [[04_interfaces](04_interfaces.md#UI-3)

---

## 3.4 FR-4: Генерация IfcPropertySetTemplate
**Описание**: Система создаёт объекты `IfcPropertySetTemplate` в рамках `IfcProjectLibrary` с использованием `ifcopenshell.template`.

**Вход**: Подтверждённый список PSet
**Выход**: Валидный IFC-файл (IFC4) с библиотекой шаблонов

**Ссылки**: [[06_rules](06_rules.md), [[07_ifc_mapping](07_ifc_mapping.md), [[05_constraints](05_constraints.md#TECH-2)

---

## 3.5 FR-5: Логирование процесса
**Описание**: Система отображает лог анализа: какие требования найдены, сгруппированы, сгенерированы.

**Вход**: Данные процесса анализа и генерации
**Выход**: Текстовый лог в UI

**Ссылки**: [[04_interfaces](04_interfaces.md#UI-4)

---

## 3.6 FR-6: Скачивание IFC-файла
**Описание**: Система предоставляет пользователю сформированный IFC-файл для скачивания.

**Вход**: Сгенерированный IFC в памяти
**Выход**: Файл `.ifc` для загрузки пользователем

**Ссылки**: [[04_interfaces](04_interfaces.md#UI-2), [[05_constraints](05_constraints.md#TECH-3)
