# 9. Сценарии использования (Use Cases)

## UC-1: Загрузка IDS-файлов

**Актор**: BIM-координатор

**Предусловие**: Пользователь открыл приложение в браузере

**Основной поток**:
1. Пользователь перетаскивает `.ids` файл(ы) в зону загрузки
2. Система отображает имя файла и индикатор обработки
3. Система парсит XML IDS
4. Система отображает список загруженных файлов

**Альтернатива**:
- 2а. Пользователь кликает «Выбрать файлы» → открывается диалог
- 3а. Ошибка парсинга → система отображает сообщение об ошибке

**Постусловие**: IDS загружен, данные в памяти

**Ссылки**: [[03_features](03_features.md#FR-1), [[08_ui_mockups](08_ui_mockups.md#81)

---

## UC-2: Просмотр найденных PSet

**Актор**: Инженер-проектировщик

**Предусловие**: IDS загружен и распарсен

**Основной поток**:
1. Система отображает дерево Entity → PSet → Свойства
2. Пользователь разворачивает нужные ветки
3. Система показывает детали свойств (тип, cardinality, ограничения)

**Альтернатива**:
- 1а. PSet не найдены → сообщение «Требования к PSet не найдены»

**Постусловие**: Пользователь подтверждает генерацию

**Ссылки**: [[03_features](03_features.md#FR-3), [[08_ui_mockups](08_ui_mockups.md#82)

---

## UC-3: Генерация IFC-библиотеки

**Актор**: Разработчик САПР

**Предусловие**: PSet подтверждены пользователем

**Основной поток**:
1. Пользователь нажимает «Сгенерировать IFC»
2. Система создаёт `IfcProjectLibrary`
3. Система создаёт `IfcPropertySetTemplate` для каждого PSet
4. Система создаёт `IfcSimplePropertyTemplate` для свойств
5. Система создаёт `IfcPropertyEnumeration` для перечислений
6. Система выполняет валидацию
7. Система отображает результат и лог

**Альтернатива**:
- 6а. Ошибка валидации → система отображает ошибки в логе

**Постусловие**: IFC-файл готов к скачиванию

**Ссылки**: [[03_features](03_features.md#FR-4), [[08a_result_mockups](08a_result_mockups.md)
