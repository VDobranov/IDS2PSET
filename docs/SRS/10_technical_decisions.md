## 10.2 Обработка IDS

**Принцип:** 1 IDS → 1 IFC

- Каждый IDS обрабатывается отдельно
- Пользователь выбирает, какие IDS генерировать
- Результат: один IFC файл на каждый IDS

**Группировка внутри IDS:**
- Ключ группировки: **только `propertySet Name`**
- Имя PSet — точно как в IDS (`propertySet/simpleValue`)
- `ApplicableEntity` объединяет все entity из требований

**Пример:**
```
IDS содержит:
  Specification "Таблица 4.А.3" (IfcWall)
    └─ PSet "Местоположение" → 3 свойства
  Specification "Таблица 4.А.1" (IfcBuilding)
    └─ PSet "Местоположение" → 2 свойства

Результат: 1 PSet "Местоположение" (5 свойств)
  ApplicableEntity: "IfcWall,IfcBuilding"
```

**Важно:** IFC не допускает разных PSet с одинаковыми именами в одной библиотеке.

## 10.3 Фильтрация PSet

- Дерево PSet содержит чекбоксы для каждого PSet
- По умолчанию все чекбоксы отмечены
- Пользователь может снять ненужные PSet
- Генерация только для отмеченных

---

## 10.4 Именование выходного файла

Формат: `{имя_IDS}_PSet_Library.ifc`

**Пример:**
- Вход: `СПб ГАУ ЦГЭ (КР).ids`
- Выход: `СПб ГАУ ЦГЭ (КР)_PSet_Library.ifc`
